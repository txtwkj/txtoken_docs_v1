## 1.1 典型场景：企业管理员为员工分配 API Key

**案例：某科技公司「云天科技」**

| 角色 | 人员 | 职责 |
|------|------|------|
| 平台运营 | 天下同网运维 | 开通企业主账号、充值/分配账户额度、配置可用模型与分组 |
| 企业管理员 | 云天科技 IT 负责人 **张三** | 登录控制台，按项目/员工创建令牌并分发 |
| 使用者 | 开发 **李四**、测试 **王五** | 在代码或工具中配置 `Base URL` + `sk-xxx`，调用 `/v1/chat/completions` 等接口 |

**推荐模式（单账号多令牌）**

```
企业主账号（张三登录）
  ├─ 令牌 dev-backend-li4      → 交给后端李四（开发环境）
  ├─ 令牌 qa-automation-wang5  → 交给测试王五（自动化脚本）
  └─ 令牌 prod-agent-service   → 交给生产 Agent 服务（限定 IP + 模型白名单）
```

- **账户额度**在「企业主账号」上统一扣减；各令牌可单独设置**额度上限**、**过期时间**、**模型限制**、**IP 白名单**。
- 若需**按人独立账单/权限**，可由平台管理员为每位成员**单独开户**（见 §1.2 方式 B），成员自行登录创建令牌。

## 1.2 注册与登录平台账号

### 方式 A：自助注册（站点已开放注册时）

1. 浏览器打开平台地址，进入 **注册** 页。
2. 填写 **用户名**、**密码**（通常 8–20 位），按页面要求完成 **邮箱验证码** 或 **人机验证**（若运营已开启）。
3. 注册成功后 **登录**；若站点开启了「注册赠送初始令牌」，可在 **令牌** 页看到系统自动创建的初始 Key（名称类似「xxx的初始令牌」）。

<Callout type="info">
是否开放注册、是否赠送初始额度，以贵司平台运营配置为准。
</Callout>

### 方式 B：由平台管理员创建企业账号（常见）

适用于企业首次接入、需预先分配 **用户分组**（如 `vip`、`default`）与 **账户额度** 的场景：

1. 联系天下同网平台管理员，提供企业名称、管理员用户名等信息。
2. 管理员在 **用户管理** 中 **添加用户**，设置初始密码与 **分组**、**额度**。
3. 企业管理员使用下发的 **用户名 + 密码** 首次登录，建议立即在 **个人设置** 中修改密码。

### 登录后您需要确认的信息

| 项目 | 在控制台中的位置 | 说明 |
|------|-----------------|------|
| **Base URL** | 首页 / 文档 / 运营通知 | API 根地址，形如 `http(s)://<平台域名>`，调用时通常拼接 `/v1/...` |
| **账户剩余额度** | 侧栏 **钱包** / **控制台** | 账户级余额；所有令牌消耗从此扣减 |
| **可用分组** | **令牌** 创建页的「令牌分组」下拉 | 决定计费倍率与可路由渠道；部分分组仅管理员可分配 |

## 1.3 创建 API 令牌

1. 使用企业管理员账号 **登录** 控制台。
2. 进入侧栏 **令牌**（令牌管理）页面。
3. 点击 **添加令牌** / **创建新的令牌**，按业务填写下表。

| 配置项 | 建议（本案例） | 说明 |
|--------|----------------|------|
| **名称** | `dev-backend-li4`、`qa-automation-wang5` | 便于审计；建议含环境 + 负责人缩写 |
| **令牌分组** | 与账户一致或选 `auto` | 影响计费倍率；`auto` 可开启 **跨分组重试** |
| **过期时间** | 开发令牌选「一个月」；生产选具体日期或「永不过期」 | 到期后 Key 立即失效 |
| **新建数量** | `1`；批量建测试 Key 可填 `5` 等 | 批量创建时名称会自动加随机后缀 |
| **无限额度** | 开发可开；生产建议 **关闭** 并设令牌上限 | 令牌额度 ≤ 账户剩余额度 |
| **金额 / 额度** | 如为李四分配等价 $50 的额度 | 用尽后该 Key 不可用，账户有余额也不透支该 Key |
| **模型限制列表** | 生产 Agent 可只勾选 `gpt-4o`、`kimi-k2.5` | 留空表示允许账户可用的全部模型 |
| **IP 白名单** | 生产服务填 office 出口 IP 或 K8s 网段 | 一行一个；留空不限制（需配合网关加固） |

4. 点击 **提交**。提示「令牌创建成功，请在列表页面点击复制获取令牌」。
5. 在令牌列表中找到对应行，点击 **复制**（可选 **仅密钥** 或 **名称+密钥**）。

<Callout type="warn">
**密钥仅在创建后首次复制时完整展示**，请当场保存；列表中默认脱敏显示。
</Callout>

**张三为本案例创建的两条令牌示例：**

| 名称 | 分组 | 过期 | 额度 | 模型限制 | 交给 |
|------|------|------|------|---------|------|
| `dev-backend-li4` | `default` | 30 天 | 无限（受账户余额约束） | 无 | 李四 |
| `qa-automation-wang5` | `default` | 30 天 | 等价 $20 | 仅 `gpt-4o-mini` | 王五 |

## 1.4 将令牌分发给员工

交给员工的说明模板（可复制）：

```
【天下同网模型 API — 开发环境】
Base URL：http://<平台地址>/v1
API Key：sk-xxxxxxxxxxxxxxxx（请勿外传）

示例（curl）：
curl http://<平台地址>/v1/chat/completions \
  -H "Authorization: Bearer sk-xxxxxxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-4o-mini","messages":[{"role":"user","content":"你好"}]}'

注意：
1. Key 只用于服务端或受信环境，不要写进前端仓库；
2. 本 Key 仅限你个人/本项目使用，泄露请立即联系管理员作废；
3. 403/401 请联系管理员检查模型权限或是否过期。
```

**OpenAI SDK 配置示例（交给李四）：**

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://<平台地址>/v1",
    api_key="sk-xxxxxxxxxxxxxxxx",  # 管理员分发的 Key
)

resp = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "你好"}],
)
print(resp.choices[0].message.content)
```

## 1.5 令牌管理与安全建议

| 操作 | 路径 | 说明 |
|------|------|------|
| **查看用量** | 控制台 **日志** / **用量** | 按令牌、模型、时间筛选 |
| **编辑令牌** | **令牌** → 编辑 | 可改额度、过期时间、IP、模型限制；**不能**查看已丢失的完整 Key |
| **作废令牌** | **令牌** → 删除 | 员工离职或 Key 泄露时立即删除 |
| **轮换 Key** | 新建令牌 → 通知员工更换 → 删除旧令牌 | 推荐定期轮换生产 Key |

**管理员实践建议：**

- **一人一 Key（或一项目一 Key）**，禁止多人共用一个生产 Key，便于追责与吊销。
- 生产环境配合 **IP 白名单** + **模型白名单** + **令牌额度上限**。
- 开发/测试/生产使用 **不同名称前缀** 与 **不同过期策略**。
- 员工 **不需要** 控制台账号时，只分发 `sk-` 即可；需要自查用量时再为其开通只读或独立子账号。
